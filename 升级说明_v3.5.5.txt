 Tangzon v3.5.5 — 待分配区修复 + 导入简化
═══════════════════════════════════════════════════

【1. 修复待分配区布局错乱（你截图反馈的问题）】
 之前：
  · 描述文字"导入但还未归类的产品..."飘到屏幕中间偏右
  · 导入产品后，产品卡位置奇怪，要下滑才能看到一角
 根因：
  · .dash-hd 是 flex 布局 + justify-content:space-between
  · 描述 <p> 作为同级元素被推到最右边
 修复：
  · 重构布局，标题区和描述分开
  · 现在产品卡正常出现在描述下方，紧凑显示

【2. 导入到待分配区 — 只需要 ASIN】
 之前：模板要求多列才能识别
 现在：
  · 只需要文件首行有 "ASIN" 列
  · 其他列都是**可选**：
      sku / 新货号 / 价格 / 星级 / 评论数
      大类 / 小类 / 备注 / 等级
  · 有就读取，没有就跳过

【3. 分配时智能预选原始类目】
 如果你导入的文件本来就带有 "大类" 和 "小类" 列，
 那分配产品时弹窗会自动预选那个类目：
  · 标题旁会显示 "来自文件提示：户外野营 › 遮阳伞"
  · 大类小类下拉自动选好
  · 你只要按"确认分配"就行（不用再手选）
 适合推广给其他卖家时使用 — 他们的文件如果带类目，
 一键就能正确归位。

【升级方法】
 5 个文件复制到 tangzon-app 覆盖 →
 GitHub Desktop Commit (v3.5.5) → Push → 自动更新
